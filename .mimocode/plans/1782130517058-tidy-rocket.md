# Plan: New Appointment Flow — Oral Medicine First

## Current Flow
```
Patient books → General Doctor (DNT02) → DNT02 fills General Case Sheet → Selects Dept → PG/UG Assigned via RR
```

## New Flow
```
Patient books → Oral Medicine Doctor → PG/UG Auto-assigned (RR) → PG fills Oral Medicine Case Sheet → Enters Treatment Plan + Department → Forwards to Dept → New PG/UG Assigned (RR)
```

## Step 2: Route Appointments to Oral Medicine Doctor + Auto-Assign PG

**File:** `server/routes/appointment.js`

### 2a. Narrow `getAssignableDoctors()` (line 220-239)

Currently filters all doctors in `GENERAL_DOCTOR_DEPARTMENT_KEYS`. Change to only return doctors whose normalized department matches `oralmedicineandradiology`:

```js
const ORAL_MEDICINE_DEPT_KEYS = new Set(['oralmedicine', 'oralmedicineandradiology', 'oralmedicineradiology']);

const getAssignableDoctors = async () => {
  const doctors = await User.find(
    { role: "doctor" },
    { _id: 1, name: 1, Identity: 1, role: 1, department: 1 }
  ).lean();

  const oralMedicineDoctors = doctors.filter((doctor) => {
    return ORAL_MEDICINE_DEPT_KEYS.has(normalizeDepartment(doctor.department));
  });

  oralMedicineDoctors.sort(/* same as current */);
  return oralMedicineDoctors;
};
```

### 2b. Modify `pickDoctorForSlot()` (line 263) to also return a PG

After picking the Oral Medicine doctor, import and call `pickPgForDoctor` from referralFlow.js to get a PG/UG from the Oral Medicine department:

## Step 4: Disable General Case Sheet for Doctor-Initiated Saves

**File:** `server/routes/general-case.js`

### Modify validation at line 348-356

Currently, doctors in general departments CAN create General Case Sheets. Change to block ALL doctor-initiated saves (only PGs/UGs can create referrals in the new flow):

```js
// OLD: Only blocks non-general doctors
if (requesterRole === 'doctor' && !GENERAL_DEPARTMENT_KEYS.has(requesterDepartment)) {
  return res.status(403).json({ ... });
}

// NEW: Block ALL doctor-initiated saves; only PGs/UGs can create referral case sheets
if (requesterRole === 'doctor') {
  return res.status(403).json({
    success: false,
    message: 'Doctors cannot create referral case sheets. Only PGs/UGs can create referrals after filling the Oral Medicine case sheet.'

```

**Note:** The existing General Case Sheet component (`Generalcasesheet.jsx`) will still be accessible but submissions will be blocked server-side. ## End-to-End Flow Verification

### Step A: Patient Books Appointment
1. Patient books via SlotBooking → `POST /api/appointment/appointments`
2. `pickDoctorForSlot()` picks Oral Medicine doctor (round-robin)
3. `pickPgForDoctor()` picks PG/UG from Oral Medicine department (round-robin)
4. Appointment created with: `doctorId = OM Doctor`, `assignedPgUgId = OM PG`, `status = "assigned"`

### Step B: PG Sees Appointment
1. PG logs into PG Dashboard → My Appointments
2. `GET /api/appointment/pg-appointments` → finds appointment via `assigned_pg_ug_id` match
3. ✅ Appointment appears

### Step C: Doctor Sees Appointment
1. Oral Medicine doctor logs into Doctor Dashboard
2. My Appointments → sees all assigned appointments (general doctor view)
3. PG Appointments → initially empty (no GeneralCase yet); appears after step D

### Step D: PG Fills Oral Medicine Case Sheet
1. PG opens Oral Medicine form (`/oral-medicine`)
2. Fills 8 pages of case sheet
3. On Page 7: enters treatment plan, selects referral department(s), adds digital signature
4. Submits → `POST /api/oral` (saves oral case) + `POST /api/general/save` (creates referral)
5. `POST /api/general/save`:
   - Picks specialist doctor in referred department (RR)
   - Picks PG/UG in referred department (RR)
   - Creates GeneralCase with referral info
   - Updates appointment: `assignedPgUgId = new PG`, `supervisingDeptDoctorId = new specialist`

### Step E: Appointment Appears in New Department
1. New PG logs in → My Appointments → sees the appointment
2. New department's doctor → PG Appointments → sees the referral
3. New department's doctor → My Appointments → sees the appointment (via `supervisingDeptDoctorId`)
