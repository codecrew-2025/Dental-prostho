# Plan: Department-Specific Case History

## Current Behavior
`caseHistory.jsx` fetches case sheets from ALL department endpoints simultaneously (pedodontics, complete-denture, fpd, implant, partial, oral, general) and shows everything.

## Desired Behavior
Case history should only show cases from the current user's department:
- Oral Medicine doctor/PG → only oral case sheets
- Prosthodontics doctor/PG → only FPD, Implant, Partial, Complete Denture cases
- Pedodontics doctor/PG → only pedodontics cases
- etc.

## Implementation

**File**: `client/src/pages/caseHistory.jsx`

1. Get the user's department from `user.department` or `localStorage.getItem('pgDepartment')` / `localStorage.getItem('doctorDepartment')`
2. Map the department to the relevant API endpoints:
   - `Oral Medicine and Radiology` → `/api/oral/patient/{id}` only
   - `Prosthodontics` → `/api/fpd/patient/{id}`, `/api/implant/patient/{id}`, `/api/ImplantPatient/patient/{id}`, `/api/partial/patient/{id}`, `/api/complete-denture/patient/{id}`
   - `Pedodontics` → `/api/pedodontics/patient/{id}` only
   - Other departments → their specific endpoint(s)
   - Unknown/general → keep the current behavior (show all)
3. Add a department filter dropdown at the top so user can override if needed
4. Show "All" option to see everything (current behavior)

## Verification
1. Login as Oral Medicine PG → open case history → only oral cases shown
2. Login as Prosthodontics PG → open case history → only prostho cases shown
3. Select "All" from dropdown → shows everything
